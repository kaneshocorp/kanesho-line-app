# LINE公式アカウント セットアップ手順

金山商店のLINE公式アカウントを、このアプリ（kanesho-line-app）と連携させるための手順書です。
スタッフの方がこの手順書だけで設定できるように書いています。上から順番に進めてください。

---

## 0. 事前準備：Googleアカウント

LINE公式アカウントの作成にはGoogleアカウントでのログインが必要です。
**金山商店名義のGoogleアカウント**を用意してください（個人アカウントでの作成は避け、退職・異動時の引き継ぎが困難にならないようにするためです）。

---

## 1. LINE公式アカウントの作成

1. [LINE Official Account Manager](https://manager.line.biz/) にアクセスし、金山商店名義のGoogleアカウントでログインします。
2. 画面の案内に従って新規アカウントを作成します。アカウント名・業種・所在地などの基本情報を入力してください。
3. 作成が完了すると、管理画面（LINE Official Account Manager）にログインできるようになります。

### Messaging APIの有効化

1. LINE Official Account Managerの管理画面で「設定」→「Messaging API」を開きます。
2. 「Messaging APIを利用する」を選択し、案内に従ってプロバイダー（会社/組織の単位。無ければ新規作成）を設定します。
3. 有効化が完了すると、同じアカウントが [LINE Developers コンソール](https://developers.line.biz/console/) 上のチャネルとしても表示されるようになります。

---

## 2. LINE Developersコンソールでの設定

1. [LINE Developers コンソール](https://developers.line.biz/console/) に、同じGoogleアカウントでログインします。
2. 該当のプロバイダー内にある、先ほど作成したMessaging APIチャネルを開きます。

### チャネルシークレットの取得

1. チャネルの「チャネル基本設定」タブを開きます。
2. 「チャネルシークレット」の値をコピーします。これが `LINE_CHANNEL_SECRET` になります。

### チャネルアクセストークン（長期）の取得

1. チャネルの「Messaging API設定」タブを開きます。
2. 「チャネルアクセストークン（長期）」の項目で「発行」をクリックします。
3. 発行された値をコピーします。これが `LINE_CHANNEL_ACCESS_TOKEN` になります。
   - この値は再表示できないことがあるため、発行時に必ず控えておいてください。
   - 万一紛失した場合は「再発行」で新しいトークンを発行し、`.env.local` とデプロイ先（Vercel等）の環境変数を両方更新してください。

### Webhook URLの設定

1. 同じ「Messaging API設定」タブの「Webhook設定」欄で、「Webhook URLを編集」をクリックします。
2. 以下のURLを設定します（`<デプロイ先ドメイン>` は実際にデプロイしたVercel等のドメインに置き換えてください）。

   ```
   https://<デプロイ先ドメイン>/api/line/webhook
   ```

3. 「Webhookの利用」を **ON** にします。
4. 「検証」ボタンを押して、200 OKが返ることを確認します（アプリが正しくデプロイされ、環境変数が設定済みである必要があります）。

### 応答メッセージ機能をOFFにする

1. 「Messaging API設定」タブの少し下、または「応答設定」欄にある「応答メッセージ」を **OFF** にしてください。
   - これはLINE公式アカウントの標準機能（自動応答）で、ONのままだとWebhook経由のこのアプリの返信処理と競合し、二重に返信されたり意図しない自動応答が返ったりします。
2. 「あいさつメッセージ」も基本的にOFFにしておくことを推奨します（friend追加時の挨拶はこのアプリのwebhookが行うため）。

---

## 3. アプリ側の環境変数設定

プロジェクトルートの `.env.local`（無ければ `.env.example` をコピーして作成）に、以下を設定してください。

```bash
# Supabase — プロジェクト設定 > API から取得
NEXT_PUBLIC_SUPABASE_URL=
NEXT_PUBLIC_SUPABASE_ANON_KEY=
SUPABASE_URL=
SUPABASE_SERVICE_ROLE_KEY=

# LINE — LINE Developers コンソール > Messaging API設定 から取得
LINE_CHANNEL_ACCESS_TOKEN=（手順2で発行した長期トークン）
LINE_CHANNEL_SECRET=（手順2で取得したチャネルシークレット）

# 公開されている価格ページのURL（Flexメッセージのボタン・リッチメニューのリンク先に使う）
NEXT_PUBLIC_SITE_URL=https://<デプロイ先ドメイン>
```

デプロイ先（Vercel等）でも同じ環境変数を設定してください。ローカルの `.env.local` はデプロイには反映されません。

---

## 4. リッチメニューのセットアップ

環境変数の設定が終わったら、以下のコマンドを一度だけ実行してください。トーク画面下部に表示される6つのボタン付きメニュー（今週の買取価格・営業カレンダー・写真でかんたん査定・店舗アクセス・品目一覧・電話で相談）が自動的に作成されます。

```bash
npm run setup:richmenu
```

- 実行結果として `richMenuId` などが表示されれば成功です。
- 画像はスクリプトが自動生成するため、デザインツールなどの準備は不要です。
- メニュー内容（ボタンの文言やリンク先）を変更したい場合は `scripts/setup-richmenu.ts` を編集し、再度このコマンドを実行してください（実行するたびに新しいリッチメニューが作成され、デフォルトメニューとして設定し直されます）。

---

## 5. 動作確認

1. LINE公式アカウントを友だち追加してみてください。
   - 「友だち追加ありがとうございます。会社名（法人のお客様のみ）を教えてください。個人のお客様は「個人」とだけ送ってください。」という返信が来ることを確認します。
2. 会社名（または「個人」）を送信し、「登録ありがとうございます。今後よろしくお願いいたします。」という返信が来ることを確認します。
3. リッチメニューの「写真でかんたん査定」をタップし、案内メッセージが来ることを確認します。
4. 実際に写真を1枚送ってみて、「写真を受け取りました。追ってご連絡いたします。」という返信が来ることを確認します。
5. 管理画面（このアプリの管理者向けページ）の写真査定タブで、送った写真がサムネイル表示されることを確認します。

うまくいかない場合は、まず以下を確認してください。

- Webhook URLの「検証」が成功しているか（LINE Developersコンソール）。
- 応答メッセージ機能がOFFになっているか。
- `.env.local` およびデプロイ先の環境変数に過不足がないか。
- デプロイ先のログ（VercelのFunction Logsなど）にエラーが出ていないか。
